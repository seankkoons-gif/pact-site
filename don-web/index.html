Why use Pact

When autonomous systems act, someone must be accountable. Pact makes that accountability provable.

Evidence, not logs

Pact produces sealed, replayable evidence bundles — not runtime logs or dashboards. What happened can be independently verified, even years later, without trusting the operator.

Responsibility without arbitration

Pact assigns responsibility using deterministic Default Blame Logic. No human review loop, no subjective interpretation, no marketplace arbitration layer.

Verifier-first trust

Trust lives in the verifier, not the vendor. Every Pact artifact can be validated offline, without network access, proprietary services, or runtime dependencies.

Institution-ready outputs

Pact produces artifacts legal, risk, and insurers already understand: PDFs, summaries, signed judgments, and audit-grade evidence packs.

Pact does not decide outcomes. It proves what happened and who was responsible.

Canonical use case: Autonomous API Procurement

Buyer agent buys a capability from Provider agent. Three outcomes.

Success

Settlement completes. Evidence bundle includes signed transcript, verification PASS, settlement confirmation.

DBL: NO_FAULT

Policy abort PACT-101

Provider violates policy (e.g. price, rate limit). Negotiation aborts before settlement.

DBL: RESPONSIBLE_ACTOR: provider

Timeout PACT-404

Provider misses delivery deadline. Settlement attempt recorded; failure event in pack.

DBL: RESPONSIBLE_ACTOR: provider
