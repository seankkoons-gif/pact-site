Pact turns agent negotiations and actions into cryptographic evidence bundles that can be verified offline, reviewed by legal/risk, and used to assign responsibility when things go wrong.

Problem: before and after

Before — logs / traces / screenshots

  • Not evidence. Tamperable. Not portable.
  • Payments prove nothing about intent.
  • Failures are ambiguous. Disputes need guesswork.
  • Insurance and audit are impossible.

After — Auditor Pack

  • Verifiable. Portable. Constitution-scoped.
  • Replayable. Hash-linked rounds. Signed actors.
  • Default Blame Logic (DBL) judgment.
  • GC and insurer-grade exports.

How it works

Full pipeline from intent to evidence and judgment.

Intent → Negotiation → Settlement → Auditor Pack → Offline Verify → DBL Judgment → GC / Insurer exports

Evidence Viewer

DBL Judgment

Exports

GC PDF. Insurer summary. Auditor-ready.

Canonical use case: Autonomous API Procurement

Buyer agent buys a capability from Provider agent. Three outcomes.

Success

Settlement completes. Evidence bundle includes signed transcript, verification PASS, settlement confirmation.

DBL: NO_FAULT

Policy abort PACT-101

Provider violates policy (e.g. price, rate limit). Negotiation aborts before settlement.

DBL: RESPONSIBLE_ACTOR: provider

Timeout PACT-404

Provider misses delivery deadline. Settlement attempt recorded; failure event in pack.

DBL: RESPONSIBLE_ACTOR: provider
