Verifier

The verifier validates Pact transcripts and evidence bundles. It checks signatures, hash chains, and policy compliance.

Verification is deterministic: same input, same output. No subjectivity.

What the verifier checks

  1. Signature validity: Every round must be signed by all agents. Invalid or missing signatures fail verification.
  2. Hash chain integrity: Each round must reference the previous round's hash. A broken chain fails verification.
  3. Policy hash match: The policy executed must match the declared policy hash.
  4. Constitution hash match: The constitution applied must match the declared hash.
  5. Schema version: Transcript and bundle must use supported schema versions.

Verification output

The verifier emits a binary result:

  • PASS: All checks valid. Evidence is admissible.
  • FAIL: One or more checks invalid. Evidence is not admissible. Specific failure reasons are provided.

CLI usage

# Verify a transcript
pact replay --transcript ./transcript.json

# Verify an evidence bundle
pact verify --bundle ./evidence_bundle.json

# Output format: JSON
pact replay --transcript ./transcript.json --format json

Verifier properties

  1. Deterministic: Same input always produces the same output.
  2. Standalone: Requires no network access, no external dependencies beyond the evidence bundle.
  3. Auditor-friendly: No SDK or wallet required. Can be run by legal, risk, or audit teams.
  4. Fast: Verification completes in milliseconds.

Who runs the verifier

Anyone can run the verifier on a Pact evidence bundle:

  • Auditors validating agent transactions
  • Risk teams investigating failures
  • Legal teams in dispute resolution
  • Insurers underwriting autonomous systems
  • Third parties conducting independent verification

Tamper detection

Any modification to a signed transcript breaks the hash chain. The verifier detects this and fails verification.

This property makes evidence bundles tamper-evident: alterations are cryptographically detectable.
