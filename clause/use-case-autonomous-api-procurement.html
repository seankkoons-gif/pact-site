Use Case

Autonomous API Procurement

An AI agent needs to purchase compute time, API access, or data services from multiple providers. It must negotiate price, enforce SLAs, detect failures, and prove what happened—all autonomously.

Discovery → Negotiation → Settlement → Evidence → Attribution

The scenario

A company deploys an autonomous agent that needs to:

  • Discover available API providers (compute, data, models)
  • Negotiate pricing and service-level agreements
  • Execute purchases and trigger payments
  • Detect and classify failures
  • Produce audit trails for finance, legal, and risk teams

Without Pact

Problems

  • Agent logs are mutable and not admissible as evidence
  • Payments happen, but intent and agreement are unclear
  • When a provider fails SLA, there's no cryptographic proof
  • Disputes require manual investigation and guesswork
  • Risk teams cannot underwrite or approve autonomous spending

With Pact

Solution

The agent wraps high-risk actions inside Pact. Every negotiation round is signed, hash-linked, and policy-enforced. Settlement is atomic. Evidence is automatic.

Step 1: Intent declaration

The agent declares what it needs:

{
  "intent": "purchase_compute",
  "budget": "≤ $50",
  "latency": "≤ 200ms",
  "reputation_min": 0.90,
  "policy": "policy_hash_abc123"
}

Step 2: Negotiation (signed rounds)

Providers respond. Each round is signed by both parties:

  • Provider offers: $45, 180ms latency, reputation 0.95
  • Agent accepts
  • Both parties sign the agreement

Step 3: Policy enforcement

Before settlement, Pact validates:

  • Budget constraint met ($45 ≤ $50)
  • Latency SLA met (180ms ≤ 200ms)
  • Reputation threshold met (0.95 ≥ 0.90)

If any constraint is violated, the transaction aborts before payment.

Step 4: Settlement

Payment executes. The provider delivers the service. Pact records the outcome:

  • ✅ Settlement succeeded
  • ❌ Provider missed SLA (latency was 250ms)
  • ❌ Provider failed to deliver

Step 5: Evidence bundle

Pact produces a portable evidence bundle containing:

  • Signed transcript of the negotiation
  • Policy hash and enforcement results
  • Settlement attempt records
  • Failure classification (if any)
  • Responsibility judgment (Default Blame Logic)

When something fails

If the provider misses SLA or fails to deliver:

  1. Pact identifies the failure event from the settlement record
  2. Default Blame Logic (DBL) assigns responsibility
  3. Evidence bundle is portable to legal, insurance, or arbitration

No manual investigation. No ambiguity. The transcript proves who agreed to what.

Who benefits

  • CFO/Finance: Autonomous spending has audit trails
  • Risk/Security: Policy violations are detected before payment
  • Legal: Disputes are resolved from evidence, not testimony
  • Insurers: Autonomous workflows are underwritable
  • Developers: No need to build custom dispute systems
